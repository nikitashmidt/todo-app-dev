Что такое React?
React
- это библиотека JavaScript с открытым исходным кодом, которая используется для создания составных пользовательских интерфейсов, особенно для одностраничных приложений. Он используется для обработки уровня представления для веб-и мобильных приложений на основе компонентов в декларативном подходе. React был создан
Джорданом Уолке
, инженером-программистом Facebook. React впервые был размещен в ленте новостей Facebook в 2011 году и Instagram в 2012 году.
Перечислите особенности React?
React
использует компонентный подход для создания UI, что позволяет разбить приложение на множество независимых компонентов.
Виртуальный DOM:
React использует виртуальный DOM, который позволяет ему обновлять только те части страницы, которые изменились, вместо обновления всей страницы. Это улучшает производительность и делает приложения более отзывчивыми, также еще нам в этом помогает Fiber.
JSX: React использует JSX
- расширение языка JavaScript, которое позволяет разработчикам писать HTML-подобный код внутри JavaScript. Это делает код более читаемым и понятным.
Однонаправленный поток данных:
React использует однонаправленный поток данных, который означает, что данные передаются только в одном направлении - от родительского компонента к дочернему. Это упрощает отладку и делает код более предсказуемым.
Большое сообщество: React
имеет большое сообщество разработчиков, которые создают и поддерживают множество библиотек и инструментов для упрощения разработки. Это делает React более доступным и удобным для использования.
Что такое Virtual DOM
Virtual DOM (виртуальный DOM)
- это концепция, которая заключается в создании в памяти браузера виртуальной копии реального DOM-дерева. Виртуальный DOM представляет собой объект, который содержит информацию о текущем состоянии элементов на странице и их свойствах. Когда происходят изменения в компонентах React, React создает новое дерево виртуального DOM, сравнивает его с предыдущим состоянием и определяет, какие изменения необходимо внести в реальный DOM. Затем React обновляет только те элементы, которые изменились, вместо полной перерисовки всего дерева. Это позволяет уменьшить количество операций, которые необходимо выполнить при обновлении UI, и значительно повысить производительность приложения. В React виртуальный DOM используется для того, чтобы обновлять UI максимально эффективно и минимизировать количество операций с реальным DOM. Кроме того, использование виртуального DOM позволяет разработчикам создавать более сложные компоненты и приложения, не беспокоясь о производительности.
Fiber
Fiber
— это новый механизм согласования или повторная реализация основного алгоритма в React v16. Цель React Fiber — повысить его пригодность для таких областей, как анимация, макет, жесты, возможность приостанавливать, прерывать или повторно использовать работу, а также назначать приоритет различным типам обновлений; и новые примитивы параллелизма.
Реконсиляция
- это процесс сравнения предыдущего и текущего состояния компонентов React и определения, какие изменения необходимо внести в пользовательский интерфейс. Ранее в React использовался стек вызовов (call stack) для реконсиляции компонентов, что могло приводить к блокировке браузера и плохой производительности при работе с большими и сложными приложениями. Fiber же представляет собой новый алгоритм реконсиляции, который использует асинхронные вычисления и позволяет React более гибко управлять процессом реконсиляции компонентов. Fiber разбивает процесс реконсиляции на множество мелких задач, которые могут быть выполнены по частям, без блокировки браузера. Это позволяет React более точно контролировать процесс реконсиляции и улучшить производительность при работе с большими и сложными приложениями. Кроме того, Fiber также предоставляет новые возможности для оптимизации производительности, такие как приоритеты задач и возможность приостанавливать и возобновлять процесс реконсиляции. Это делает React более гибким и удобным для разработки сложных пользовательских интерфейсов.
Для чего нужен атрибут key при рендере списков?
Атрибут key используется при рендере списков в React для оптимизации производительности и избежания проблем с обновлением элементов списка. Когда React рендерит список элементов, каждый элемент должен иметь уникальный ключ (key), который помогает React определить, какие элементы были изменены, добавлены или удалены.
Методы жизненного цикла
Методы жизненного цикла
компонента в React - это методы, которые вызываются в определенные моменты жизненного цикла компонента, такие как создание, обновление, монтирование и размонтирование. Вот список методов жизненного цикла компонента в React:
1. Mounting (Монтирование)
- это стадия, когда компонент создается и добавляется в DOM. В этой стадии вызываются следующие методы:
- constructor()
- static getDerivedStateFromProps()
- render()
- componentDidMount()
2. Updating (Обновление)
- это стадия, когда компонент обновляется в ответ на изменение его пропсов или состояния. В этой стадии вызываются следующие методы:
- static getDerivedStateFromProps()
- shouldComponentUpdate()
- render()
- getSnapshotBeforeUpdate()
- componentDidUpdate()
3. Unmounting (Размонтирование)
- это стадия, когда компонент удаляется из DOM. В этой стадии вызывается следующий метод:
- componentWillUnmount
4. Error Handling (Обработка ошибок)
- это стадия, которая вызывается, когда в компоненте происходит ошибка. В этой стадии вызываются следующие методы:
- static
- getDerivedStateFromError()
- componentDidCatch()
Каждый метод жизненного цикла компонента выполняет определенную задачу и может быть использован для реализации различных функциональностей в компонентах React.
